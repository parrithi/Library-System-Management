## 📖 Project Overview

Managing a library involves much more than storing book records. Libraries must efficiently track book availability, member registrations, borrowing history, returns, employee activities, overdue books, and rental revenue.

This project simulates a real-world **Library Management Database** built entirely using **PostgreSQL**, focusing on database design, transaction management, and business analytics.

The project demonstrates practical SQL skills commonly required in **Data Analyst**, **Business Analyst**, **SQL Developer**, and **Database Developer** roles.

---

# 🎯 Business Problem

Traditional library systems often struggle to answer important business questions such as:

* Which books are most frequently borrowed?
* Which members have overdue books?
* How much rental revenue is generated by each category?
* Which employees process the most book issues?
* Which branches perform the best?
* Which books are currently unavailable?
* Which members are actively using the library?

This project solves these challenges using SQL-driven reporting and database automation.
